# 향후 기획 방향성 가이드

## 개요

이 문서는 게임 개발의 단계별 확장 계획과 기획 방향성을 제시합니다. 초기 MVP(Minimum Viable Product)를 완성한 후, 단계적으로 기능을 확장해 나가는 전략을 다룹니다.

---

## 1. MVP (최소 기능 제품) 정의

### 1.1 필수 기능
다음 기능들이 완성되어야 최소한의 게임으로 플레이 가능합니다:

#### 클라이언트
- [ ] 캐릭터 자동 이동 (횡스크롤)
- [ ] 기본 몬스터 스폰 및 AI
- [ ] 기본 전투 시스템 (공격/피격)
- [ ] 체력 시스템
- [ ] 골드 획득 및 표시
- [ ] 무기 1종류 구현
- [ ] 무기 강화 UI 및 로직
- [ ] 기본 게임 오버/재시작

#### 서버
- [ ] 사용자 인증 (회원가입/로그인)
- [ ] 플레이어 데이터 저장/불러오기
- [ ] 무기 데이터 저장/불러오기
- [ ] 기본 API 엔드포인트

### 1.2 MVP에서 제외할 기능 (후순위)
- Apple Watch 연동 (Phase 7로 이동)
- 이벤트 시스템 (Phase 6로 이동)
- 던전 시스템 (Phase 5로 이동)
- 복잡한 UI/애니메이션

---

## 2. 단계별 확장 전략

### Phase 1: MVP 완성 (우선순위 최고)
**목표**: 플레이 가능한 최소한의 게임 완성

**핵심 작업**:
1. 기본 게임플레이 루프 구현
2. 간단한 전투 시스템
3. 기본 강화 시스템
4. 서버 연동 (데이터 저장)

**성공 기준**:
- 게임을 시작하고 몬스터를 처치할 수 있음
- 골드를 모아 무기를 강화할 수 있음
- 게임을 종료하고 다시 시작해도 진행 상황이 저장됨

---

### Phase 2: 게임플레이 심화
**목표**: 게임의 재미 요소 강화

**추가 기능**:
- 다양한 무기 타입 (검, 활, 마법봉)
- 무기별 고유 공격 패턴
- 몬스터 종류 다양화 (3-5종)
- 레벨 시스템 (경험치 획득 및 레벨업)
- 스킬 시스템 (액티브/패시브 스킬)

**기획 고려사항**:
- 각 무기 타입의 밸런스 조정
- 몬스터 난이도 곡선 설계
- 레벨업 보상 설계

---

### Phase 3: 던전 시스템
**목표**: 게임의 목표와 구조 제공

**추가 기능**:
- 스테이지 시스템 (1-1, 1-2, ...)
- 던전 입장/클리어 시스템
- 보스 몬스터
- 던전별 특별 보상

**기획 고려사항**:
- 던전 난이도 곡선
- 보스 전투 패턴 설계
- 클리어 보상 밸런스

---

### Phase 4: 이벤트 시스템
**목표**: 게임의 지속적인 재미 제공

**추가 기능**:
- 시간 제한 이벤트
- 로그인 보너스
- 특별 몬스터 출현 이벤트
- 서버에서 이벤트 관리

**기획 고려사항**:
- 이벤트 빈도 및 지속 시간
- 이벤트 보상 설계
- 이벤트 알림 시스템

---

### Phase 5: Apple Watch 연동
**목표**: 플랫폼 확장 및 사용자 경험 향상

**추가 기능**:
- Watch에서 게임 상태 확인
- Watch에서 간단한 액션 수행
- Watch 알림
- Watch 전용 미니 게임 (선택)

**기획 고려사항**:
- Watch의 제한된 화면 크기 고려
- 배터리 소모 최소화
- Phone-Watch 동기화 전략

---

## 3. 기획 시 고려사항

### 3.1 게임 밸런스

#### 난이도 곡선
- **초반 (1-10레벨)**: 쉬운 난이도로 학습 유도
- **중반 (10-30레벨)**: 점진적 난이도 증가
- **후반 (30레벨+)**: 도전적 난이도, 강화 필요

#### 경제 시스템
- **골드 획득량**: 몬스터 처치 시 적절한 보상
- **강화 비용**: 지수적 증가로 장기 목표 제공
- **인플레이션 방지**: 골드 획득과 소비의 균형

### 3.2 사용자 경험 (UX)

#### 방치형 게임 특성
- **오프라인 진행**: 게임을 꺼도 일정 시간 진행 (선택)
- **자동 진행**: 사용자가 직접 조작하지 않아도 진행
- **간단한 조작**: 복잡한 조작 최소화

#### 진행감 제공
- **시각적 피드백**: 레벨업, 강화 시 만족스러운 효과
- **목표 설정**: 단기/중기/장기 목표 제공
- **보상 시스템**: 작은 성취에도 보상 제공

### 3.3 수익 모델

#### 무료 플레이 모델
- **광고**: 선택적 광고 시청으로 보상
- **인앱 구매**: 
  - 시간 절약 (골드 구매)
  - 편의성 (광고 제거)
  - 컬렉션 (특별 무기/아이템)

#### 밸런스 고려
- **과금 유도 vs 공정성**: 과금 유저와 무과금 유저의 밸런스
- **페이월 방지**: 과금 없이도 충분히 즐길 수 있어야 함

## 4. 데이터 기반 기획

### 4.1 수집할 데이터
- 플레이 시간
- 레벨별 이탈률
- 무기 사용률
- 강화 빈도
- 이벤트 참여율

### 4.2 분석 지표
- **DAU/MAU**: 일일/월간 활성 사용자
- **리텐션**: 1일/7일/30일 리텐션
- **ARPU**: 유저당 평균 수익
- **게임플레이 지표**: 평균 플레이 시간, 최대 레벨

### 4.3 개선 사이클
1. **데이터 수집**: 게임 플레이 데이터 수집
2. **분석**: 문제점 및 개선점 파악
3. **가설 설정**: 개선 방안 가설 수립
4. **A/B 테스트**: 소규모 테스트
5. **전면 적용**: 성공한 개선사항 전면 적용

## 5. 컨텐츠 확장 계획

### 5.1 단기 (3-6개월)

#### 캐릭터 시스템
- **다양한 캐릭터 클래스**: 전사, 궁수, 마법사
- **캐릭터별 특성**: 고유 스킬 및 능력치
- **캐릭터 커스터마이징**: 외형 변경 (선택)

#### 펫 시스템
- **펫 획득**: 몬스터 처치 또는 구매
- **펫 육성**: 레벨업 및 진화
- **펫 능력**: 자동 공격 보조, 버프 제공

### 5.2 중기 (6-12개월)

#### 소셜 기능
- **친구 시스템**: 친구 추가 및 비교
- **길드 시스템**: 길드 생성, 가입, 길드 던전
- **리더보드**: 전역/친구 랭킹

#### PvP 시스템
- **비동기 PvP**: 상대방의 기록과 경쟁
- **실시간 PvP**: 실시간 대전 (선택)
- **시즌 시스템**: 정기적인 시즌 및 보상

### 5.3 장기 (12개월+)

#### 메타 게임
- **컬렉션 시스템**: 다양한 무기/아이템 수집
- **업적 시스템**: 다양한 업적 및 보상
- **스토리 모드**: 챕터별 스토리 진행

#### 플랫폼 확장
- **PC 버전**: Steam 등 플랫폼 출시
- **웹 버전**: 브라우저에서 플레이 가능
- **크로스 플랫폼**: 모든 플랫폼에서 동일 계정 사용

## 8. 기획 체크리스트

새로운 기능을 추가할 때 다음 사항을 확인하세요:

- [ ] 이 기능이 게임의 핵심 재미를 향상시키는가?
- [ ] 구현 복잡도 대비 효과가 충분한가?
- [ ] 기존 시스템과 충돌하지 않는가?
- [ ] 밸런스에 영향을 주는가? (영향이 있다면 조정 계획은?)
- [ ] 사용자 경험을 해치지 않는가?
- [ ] 서버 부하를 고려했는가?
- [ ] 테스트 계획이 있는가?
