# 일일 개발 계획 및 마일스톤

## 2일차 작업: Flutter + Flutter Flame 세팅

### 목표
Flutter 프로젝트 초기화 및 Flame 엔진 기본 설정

### 작업 목록

#### 1. Flutter 프로젝트 초기화
- [ ] Flutter 프로젝트 생성 (`client` 폴더)
- [ ] 프로젝트 구조 설정 (기획서 참고)
- [ ] 기본 의존성 추가 (`pubspec.yaml`)
  - `flame` (최신 버전)
  - `provider` 또는 `riverpod` (상태 관리)
  - `dio` 또는 `http` (API 통신)
  - `shared_preferences` (로컬 저장소)

#### 2. Flame 엔진 기본 설정
- [ ] `main.dart`에서 Flame 게임 초기화
- [ ] 기본 게임 클래스 생성 (`Game` 클래스 상속)
- [ ] 화면 크기 및 해상도 설정
- [ ] 기본 게임 루프 확인

#### 3. 프로젝트 구조 생성
- [ ] `lib/game/` 폴더 구조 생성
- [ ] `lib/models/` 폴더 생성
- [ ] `lib/services/` 폴더 생성
- [ ] `lib/screens/` 폴더 생성
- [ ] `assets/` 폴더 생성 (images, audio)

#### 4. 기본 화면 구성
- [ ] 홈 화면 (게임 시작 화면)
- [ ] 게임 화면 (Flame 게임 표시)
- [ ] 기본 네비게이션 설정

#### 5. 개발 환경 확인
- [ ] iOS 시뮬레이터/Android 에뮬레이터에서 실행 확인
- [ ] Hot Reload 동작 확인
- [ ] 기본 로그 출력 확인

### 예상 소요 시간
- 프로젝트 초기화: 30분
- Flame 설정: 1시간
- 구조 생성: 30분
- 기본 화면: 1시간
- 테스트 및 확인: 30분
- **총 예상 시간: 3-4시간**
